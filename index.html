markdown guide

Markdown 常用语法标准指南

这是一份旨在帮助你快速掌握 Markdown 排版的参考手册。Markdown 是一种轻量级标记语言，它允许你使用易读易写的纯文本格式编写文档。

1. 标题 (Headings)

使用 # 号开头，1 到 6 个 # 分别对应一到六级标题。

一级标题 (H1)

二级标题 (H2)

三级标题 (H3)

四级标题 (H4)

五级标题 (H5)
六级标题 (H6)

2. 文本格式 (Text Formatting)

  • 加粗：**这是加粗文字** 或 __这是加粗文字__
  • 斜体：*这是斜体文字* 或 _这是斜体文字_
  • 粗斜体：***加粗且斜体***
  • 删除线：~~这是删除线~~

3. 列表 (Lists)

无序列表

使用 -、+ 或 * 加上空格。

  • 项目一
  • 项目二
    • 子项目（缩进两空格）

有序列表

使用数字加上 . 和空格。

  1. 第一步
  2. 第二步
  3. 第三步

任务列表 (Task Lists)

  • 已完成任务
  • 待办事项
  • 计划中

4. 链接与图片 (Links & Images)

  • 行内链接：[Google](https://www.google.com)
  • 带标题的链接：[百度](https://www.baidu.com "搜索引擎")
  • 插入图片：![图片描述](图片链接地址)

5. 引用与分割线 (Quotes & Rules)

引用

使用 > 符号。

这是一个块状引用。

也可以嵌套引用。

分割线

使用三个或更多的 ---、*** 或 ___。

6. 代码 (Code)

行内代码

使用单反引号包裹：print("Hello World")

代码块

使用三个反引号（```）包裹，并可以指定语言进行高亮。

def greet(name):
    return f"Hello, {name}!"

print(greet("Markdown"))

hexo guide

Hexo 常用命令手册

1. 核心流程命令 (The Big Four)

命令 简写 描述
hexo clean 无 清除缓存：删除 db.json 和 public 文件夹。建议在生成或部署前执行。
hexo generate hexo g 生成静态文件：将 Markdown 源码转化为 HTML 页面，保存在 public 文件夹。
hexo server hexo s 本地预览：启动本地服务器（默认端口 4000），用于推送到线上前查看效果。
hexo deploy hexo d 部署：将生成的静态文件推送到配置好的远程仓库分支（如 gh-pages）。

2. 内容创作命令

  • 新建文章

    hexo new post "文章标题"

  • 新建页面

    hexo new page "about"   # 创建“关于我”之类的独立页面

  • 草稿功能

    hexo new draft "草稿名" # 文件会放在 source/_drafts/，默认预览不可见
hexo publish "草稿名"   # 将草稿移动到 _posts 文件夹，正式发布

3. 配置与信息查看

  • 查看版本

    hexo version  # 检查 Hexo 环境及其插件版本

  • 列出所有路由

    hexo list post    # 查看所有文章的状态（发布、草稿等）
hexo list route   # 查看生成的页面路径（排查 404 很有用）

3. 配置与信息查看

  • -p(Port): 修改本地预览端口。如果 4000 被占用，可用 hexo s -p 5000。
  • -d (Deploy): 生成后立即部署。hexo g -d 是效率最高的方式。
  • –debug: 开启调试模式。如果命令报错，加上这个参数可以看到详细的错误堆栈信息。
